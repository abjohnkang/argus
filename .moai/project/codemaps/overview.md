# Argus — Codemaps Overview

This is a placeholder. Argus has no source code yet — codemaps will be regenerated automatically once the first SPEC is implemented and code lands in the repo.

## Planned scope (from product.md / structure.md / tech.md)

- **`api/`** (planned, Python / FastAPI): HTTP layer that fronts the local Llama 4 inference. Streams tokens to the browser. Single-user, single-process by design.
- **`web/`** (planned, React / TypeScript): Browser chat UI. Single-page app. Connects to the local `api/` over `127.0.0.1`.
- **`compose/` or root** (planned): `docker-compose.yml` orchestrating the API, web UI, and model runtime as separate containers. The container topology is one of the unresolved decisions captured in `tech.md` → Open Decisions.
- **`scripts/`** (planned): `run_server.sh`, `run_debug.sh`. Idempotent host-side launchers.
- **`.moai/`** (exists): SPEC and project documentation managed by MoAI-ADK.

## What is intentionally NOT in the planned layout

- No `cloud/`, `telemetry/`, `analytics/`, `remote/` — these conflict with the on-device privacy constraint enforced in `product.md`.
- No `auth/`, `users/`, `tenants/` — single-user is explicit. Auth is a deferred decision, not a planned module.
- No `tools/`, `agents/`, `memory/` modules in v1 — agent capabilities are deferred past the first milestone.
